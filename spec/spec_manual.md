- Source Persona: [Frequent Activity Tracker]
- Traceability: [Derived from review group G1]
- Acceptance Criteria: [Given the user is logging a workout When the logging process completes Then the application must not crash and the workout must be saved.]


Requirement ID: FR1

Description: The system should display subscription details and pricing before placing any charges on the user. Source Persona: P1 - Alex the Overcharged Subscriber Traceability: Derived from review group G1 Acceptance Criteria: Given the user is viewing the subscription page When the page loads Then the price, cycle, renewal date and how to cancel are visible to the user without navigating to another page

Requirement ID: FR2

Description: The system should allow users to cancel their subscription or free trial. Source Persona: P1 - Alex the Overcharged Subscriber Traceability: Derived from review group G1 Acceptance Criteria: Given the user would like to cancel their subscription When they navigate to Settings > Subscription Then there is a “Cancel Subscription” button that takes them to the cancel screen within the app

Requirement ID: FR3

Description: The system should not charge users after their free trial if they canceled before the trial end date. Source Persona: P1 - Alex the Overcharged Subscriber Traceability: Derived from review group G1 Acceptance Criteria: Given the user cancels their free trial before the trial end date When the trial end date arrives Then the user will not be charged for the subscription

Requirement ID: FR4

Description: The system should not crash or freeze while users are actively using any of its features. Source Persona: P2 - Jordan the Interrupted Meditator Traceability: Derived from review group G2 Acceptance Criteria: Given the user is currently playing an audio session When the user is actively using the features of the app Then the app should remain stable and not crash

Requirement ID: FR5

Description: The system should load the user’s selected content within an acceptable time frame. Source Persona: P2 - Jordan the Interrupted Meditator Traceability: Derived from review group G2 Acceptance Criteria: Given the user chooses a meditation session to play while connected to the internet When the session begins Then the audio should begin playing within 5 seconds of selection

Requirement ID: FR6

Description: The system should remain stable after being updated with the latest software updates. Source Persona: P2 - Jordan the Interrupted Meditator Traceability: Derived from review group G2 Acceptance Criteria: Given the latest update to the app has been installed and is updated on the user’s device When the user begins to use the app Then no crashes should occur with the updated app and the load time should be within 10% of the previous version of the app

Requirement ID: FR7

Description: The system should allow users to play downloaded content without being online. Source Persona: P3 - Sam the Offline Sleeper Traceability: Derived from review group G3 Acceptance Criteria: Given the user has downloaded a session from the app When the user is not connected to the internet Then the downloaded session can be played without errors

Requirement ID: FR8

Description: The system should retain any downloaded content after software updates. Source Persona: P3 - Sam the Offline Sleeper Traceability: Derived from review group G3 Acceptance Criteria: Given the user has downloaded content prior to the software update When the user updates the app Then the content will still be playable when the user is offline
